# 视觉模块 ↔ 车端 串口通信协议 V1.0

## 1. 物理层

| 参数       | 值                          |
|------------|----------------------------|
| 接口       | UART4 (LPUART4)            |
| TX 引脚    | C16                        |
| RX 引脚    | C17                        |
| 波特率     | 115200 bps                 |
| 数据位     | 8                          |
| 停止位     | 1                          |
| 校验       | 无                         |
| 连接器     | H9 (1=5V, 2=GND, 3=RX, 4=TX) |

## 2. 帧格式

```
┌────────┬────────┬─────┬─────┬──────────────┬──────────┐
│ HEADER0│ HEADER1│ CMD │ LEN │  PAYLOAD     │ CHECKSUM │
│  0xA5  │  0x5A  │ 1B  │ 1B  │  LEN bytes   │   1B     │
└────────┴────────┴─────┴─────┴──────────────┴──────────┘
```

| 字段      | 长度  | 说明                                      |
|-----------|-------|------------------------------------------|
| HEADER0   | 1B    | 固定 `0xA5`                               |
| HEADER1   | 1B    | 固定 `0x5A`                               |
| CMD       | 1B    | 命令字                                    |
| LEN       | 1B    | PAYLOAD 长度（0~200）                     |
| PAYLOAD   | LEN B | 有效载荷                                  |
| CHECKSUM  | 1B    | CMD ^ LEN ^ PAYLOAD[0] ^ ... ^ PAYLOAD[LEN-1] |

> **CHECKSUM** 采用异或校验，覆盖 CMD、LEN 和全部 PAYLOAD 字节。

## 3. 命令定义

### 3.1 地图帧 — CMD = 0x01（视觉 → 车端）

视觉模块识别到完整地图后，主动发送此帧。

**Payload 结构（共 195 字节）：**

```
┌────────┬────────┬──────────┬──────────────────────────┐
│ car_x  │ car_y  │ car_dir  │ map_data[192]            │
│  1B    │  1B    │   1B     │  192B (row-major)        │
└────────┴────────┴──────────┴──────────────────────────┘
```

| 字段     | 类型    | 范围     | 说明                                |
|----------|---------|----------|-------------------------------------|
| car_x    | uint8_t | 0 ~ 15   | 车模在地图中的 X 坐标（列号）        |
| car_y    | uint8_t | 0 ~ 11   | 车模在地图中的 Y 坐标（行号）        |
| car_dir  | uint8_t | 0 ~ 3    | 车模朝向（见下方枚举）              |
| map_data | uint8_t[192] | 0~6 | 16×12 栅格，按行优先排列            |

**car_dir 方向枚举：**

| 值 | 方向  | 说明     |
|----|-------|---------|
| 0  | UP    | 朝上(+Y) |
| 1  | RIGHT | 朝右(+X) |
| 2  | DOWN  | 朝下(-Y) |
| 3  | LEFT  | 朝左(-X) |

**map_data 单元格编码（与 grid_cell_t 一致）：**

| 值 | 类型    | 符号 | 说明         |
|----|---------|------|-------------|
| 0  | UNKNOWN | `?`  | 未知/未识别  |
| 1  | EMPTY   | `.`  | 空地，可通行 |
| 2  | WALL    | `#`  | 墙体，不可通行|
| 3  | BOX     | `B`  | 普通箱子     |
| 4  | GOAL    | `G`  | 目标点       |
| 5  | BOMB    | `X`  | 炸弹         |
| 6  | CAR     | `C`  | 车模位置     |

**map_data 排列方式：**
```
索引 0            = cells[0][0]  (row=0, col=0)
索引 1            = cells[0][1]  (row=0, col=1)
...
索引 15           = cells[0][15] (row=0, col=15)
索引 16           = cells[1][0]  (row=1, col=0)
...
索引 191          = cells[11][15](row=11, col=15)
```

### 3.2 地图请求帧 — CMD = 0x02（车端 → 视觉）

车端主动请求视觉模块发送一帧完整地图。

**Payload（1 字节）：**

| 字段         | 类型    | 说明                     |
|-------------|---------|-------------------------|
| request_type| uint8_t | 0x01=请求全量地图         |

### 3.3 心跳/ACK 帧 — CMD = 0x03（双向）

保活或应答。

**Payload（1 字节）：**

| 字段   | 类型    | 说明                         |
|--------|---------|------------------------------|
| status | uint8_t | 0x00=故障, 0x01=正常          |

## 4. 时序

```
视觉模块                              车端 (RT1064)
   │                                      │
   │ ──── [0x01 地图帧, 195B] ──────────> │  解析 → 更新 grid_map
   │                                      │
   │                       (可选请求)      │
   │ <──── [0x02 请求帧] ──────────────── │
   │                                      │
   │ ──── [0x01 地图帧] ─────────────────>│
   │                                      │
   │ ──── [0x03 心跳] ──────────────────> │  每 500ms
   │                                      │
```

- 视觉模块主动以 **5~10 Hz** 发送地图帧
- 车端可主动发送 0x02 请求帧触发立即回传
- 心跳帧可用于链路保活，超过 1000ms 未收到任何帧视为离线

## 5. 帧长度汇总

| CMD  | 方向       | PAYLOAD | 总帧长                |
|------|-----------|---------|----------------------|
| 0x01 | 视觉→车端 | 195     | 2+1+1+195+1 = 200 B  |
| 0x02 | 车端→视觉 | 1       | 2+1+1+1+1   = 6 B    |
| 0x03 | 双向      | 1       | 2+1+1+1+1   = 6 B    |
